## API Examples

### Chat API

```bash
# Basic chat
curl -X POST http://localhost:8000/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What laptops do you have?",
    "session_id": "user-123"
  }'

# Response
{
  "response": "We have several great laptops: TechPro X1 ($1,299), X2 ($1,599), UltraBook 13 ($999), and Gaming Laptop G1 ($1,899)...",
  "intent": "product_info",
  "sources": ["Product Catalog", "Product Manuals"],
  "metadata": {
    "num_results": 3
  }
}
```

### Product Endpoints

```bash
# Get all products
curl http://localhost:8000/api/products

# Get specific product
curl http://localhost:8000/api/products/1

# Compare products
curl "http://localhost:8000/api/products/compare?ids=1,2"

# Response
{
  "products": [
    {"id": 1, "name": "TechPro X1 Laptop", ...},
    {"id": 2, "name": "TechPro X2 Laptop", ...}
  ],
  "comparison": [
    {"feature": "processor", "values": {"X1": "i7", "X2": "i9"}},
    {"feature": "price", "values": {"X1": 1299.99, "X2": 1599.99}}
  ],
  "recommendation": "Based on ratings, X2 is recommended..."
}
```

### Order Endpoints

```bash
# Track order
curl http://localhost:8000/api/orders/ORD-1001

# Response
{
  "order_id": "ORD-1001",
  "customer_name": "John Smith",
  "status": "delivered",
  "items": [...],
  "total": 1299.99,
  "tracking_number": "TRK-1001234567"
}
```

### Support Endpoints

```bash
# Create complaint
curl -X POST http://localhost:8000/api/complaints \
  -H "Content-Type: application/json" \
  -d '{
    "order_id": "ORD-1001",
    "issue": "Product defect",
    "description": "Screen flickering issue"
  }'

# Response
{
  "complaint_id": "CMP-106",
  "order_id": "ORD-1001",
  "status": "open",
  "created_at": "2026-02-09 14:30:00",
  "message": "Complaint created successfully..."
}

# Check refund status
curl http://localhost:8000/api/refunds/ORD-1001

# Get delivery status
curl http://localhost:8000/api/delivery/ORD-1001
```

## Advanced Usage Patterns

### Multi-turn Conversation

```javascript
// Maintain session_id across requests
const sessionId = `session-${Date.now()}`

// Turn 1
await api.sendMessage("Show me laptops", sessionId)
// Agent: "We have X1, X2, UltraBook, Gaming..."

// Turn 2
await api.sendMessage("Tell me more about the X1", sessionId)
// Agent remembers context: "The X1 laptop I mentioned features..."

// Turn 3
await api.sendMessage("What's the warranty?", sessionId)
// Agent: "The X1 comes with a 2-year warranty..."
```
